CAYIN Index Page Redesign & Design System

設計改良-鎧應科技官網首頁及設計系統

此項專案是將舊有的鎧應科技首頁進行再設計，同時建立完整的設計系統。

負責內容 - UI/UX 設計、前端程式撰寫、設計系統建立

服務公司 - 鎧應科技股份有限公司

前往網站
UI/UX 設計 HTML / CSS javascript 響應式設計 設計系統

專案背景

CAYIN 鎧應科技是一家專注於設計數位廣告看板播放器、以及播放軟體的一家科技公司。
企業識別標誌在多年前便已修改、改用其他配色，但官網首頁仍持續使用原有設計，因此決定在公司即將推出新產品系列前進行改造，讓原先就認識或是未來新接觸的客戶都能在進入官網時有更好的體驗。

專案目標

1. 優化首頁資訊架構及 UI/UX 設計

除了更改用色之外也希望優化資訊架構及 UI/UX 設計，希望更有效的推廣產品並讓使用者留下好的印象。

2. 完整設計系統

希望藉此機會同時建立設計系統，不只是首頁，而是讓整體官網擁有統一的視覺設計、並強化企業識別。

問題提出及假設

原先的首頁宣傳橫幅是將標題文字與視覺要素做成一張圖片，我們發現此作法會影響到SEO，造成產品無法進一步推廣，因此需確保橫幅標題是獨立出來、使用網頁文字的方式。 另外，為了更有效引導客戶了解產品，在橫幅上也需要包含相關頁面連結。

客戶案例能做為良好的廣告及實際應用展示，但是在原先的首頁不僅位於頁面下方、且占有面積過小，難以有效達到宣傳目的。

設計提案

在了解問題點、並規劃出整體頁面架構後，便開始進行細部的設計，各個內容區塊通常會設計2種以上的提案。可以看到下圖中包含有特色產品以及客戶案例，前者最終以右側的設計做為定案，主要原因是: 1. 產品照片比例適中，2. 連結樣式特別，較能吸引使用者進行互動；而後者最終也是以右側的設計做為定案，主要原因是: 1. 不對稱比例設計較能吸引使用者目光停留在此區域，2. 左側能含有較多資訊。
